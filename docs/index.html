TGWrapper

Serverless-first Telegram bot framework for TypeScript. Distributed sessions, CAS consistency, structured observability — without external bot libraries.

Get Started GitHub npm

Why TGWrapper?

🔄

CAS Sessions

Compare-And-Set semantics prevent silent concurrent session overwrites across distributed instances. No last-write-wins.

📡

Serverless-first

Native adapters for AWS Lambda, Cloudflare Workers, and Node HTTP. Zero framework dependencies — pure fetch to Telegram API.

📊

Built-in Observability

Structured ECS logging, Prometheus metrics, OTLP tracing, AsyncLocalStorage correlation context — production-ready from day one.

⚡

Rate Limiting

Distributed sliding-window rate limiter (Redis ZSET-based Lua scripts) + in-memory token bucket. Anti-spam baked in.

🔌

Multi-runtime

Same code runs on Node.js, AWS Lambda, Cloudflare Workers, or any HTTP server. Polling and webhook update sources.

🔒

Circuit Breaker

Automatic failure detection and cooldown for Telegram API calls. Bounded concurrency queue prevents overload.

Quick Start

# Install
npm install @tgwrapper/core

# bot.mjs
import { createBotClient, PollingSource } from '@tgwrapper/core';

const bot = createBotClient({
  token: process.env.BOT_TOKEN,
  session: { type: 'memory' }, // no Redis needed to start
});

bot.on('message', async (ctx) => {
  await ctx.reply(`You said: ${ctx.text}`);
});

const poller = new PollingSource(bot, { token: process.env.BOT_TOKEN });
poller.start();

# Run
BOT_TOKEN=your_token node bot.mjs

Personal finance tracker with voice transcription. Multi-platform bot handling 3 surfaces (mobile, bot, API) via TGWrapper's FSM and TreeRouter.

Multi-format image conversion bot (SVG ↔ raster, OCR). Uses TGWrapper's rate limiting and bounded concurrency for fair resource allocation across users.
