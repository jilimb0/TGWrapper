# Early Advocate Program

Early advocates are the bridge between TGWrapper's core maintainers and the broader developer ecosystem. This framework defines the role, benefits, and responsibilities of advocates.

---

## 🎯 Role of an Advocate

An early advocate is an experienced TypeScript/Node.js engineer running Telegram services who believes in the TGWrapper architecture. Advocates contribute as **multipliers**:
- **Feedback Loops:** Test pre-release candidates (beta builds) and report edge integration errors.
- **Ecosystem Expansion:** Build and publish custom adapters, templates, or tutorials.
- **Support Multipliers:** Answer basic Q&A threads in GitHub Discussions to assist new developers.

---

## 🎁 Program Benefits

To appreciate their contributions, active advocates receive:
- **Direct Roadmap Input:** Direct access to core maintainers via a private Slack/Telegram channel for architectural discussions.
- **Priority PR Reviews:** Pull requests submitted by advocates are prioritized.
- **Showcase Promotion:** Advocate projects and profiles are highlighted on the main repository page and in launch announcements.

---

## 🚀 How to Participate

The Advocate Program is lightweight and invite-only. Maintainers extend invites to contributors who:
1. Submit a production pilot case study in [Discussions](https://github.com/jilimb0/tgwrapper/discussions).
2. Author an external plugin (e.g. DynamoDB/MongoDB session adapters).
3. Help answer multiple unresolved community Q&A queries.
